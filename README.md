# حاسبة عرض سعر محطة الطاقة الشمسية

موقع أونلاين بديل لملف الإكسل `1-Main_Solar_Station.xlsx` — حاسبة تفاعلية كاملة (نفس معادلات الإكسل بالظبط تم التحقق منها رقم برقم) + لوحة أدمن لتعديل الأسعار والبيانات وحفظها مباشرة على GitHub بحيث تتحدث للجميع.

## الملفات
```
index.html      ← الصفحة الرئيسية (الحاسبة + لوحة الأدمن)
style.css       ← التصميم (ثيم فاتح + خط Cairo + ستايل الطباعة)
calc.js         ← محرك الحسابات (منقول من معادلات الإكسل)
app.js          ← ربط الواجهة بالحسابات + منطق الأدمن + الحفظ على GitHub
data.json       ← كل الأسعار والبيانات (الألواح، الإنفرترات، الحديد، الكابلات...) - **ده الملف اللي الأدمن بيعدله**
assets/logo.png ← شعار الشركة (يظهر في الهيدر وفي عرض السعر المطبوع)
```

كله Static (HTML/CSS/JS بحت) — مفيش سيرفر أو قاعدة بيانات مطلوبة، شغال 100% على GitHub Pages.

**⚠️ لازم ترفع مجلد `assets` بالكامل (فيه اللوجو) مع باقي الملفات، وإلا اللوجو مش هيظهر.**

---

## 1) خطوات الرفع على GitHub Pages

1. اعمل ريبو جديد على GitHub (Public أو Private براتب Pro/Team لو عايزه Private).
2. ارفع الملفات الخمسة اللي فوق (index.html, style.css, calc.js, app.js, data.json) في جذر الريبو (أو داخل مجلد `/docs` لو حابب).
3. من إعدادات الريبو **Settings → Pages**:
   - Source: `Deploy from a branch`
   - Branch: `main` (أو الفرع اللي رفعت عليه) والمجلد `/root` (أو `/docs` لو استخدمته)
   - احفظ.
4. بعد دقيقة أو اتنين هيديك لينك زي:
   `https://<اسم-المستخدم>.github.io/<اسم-الريبو>/`
5. افتح اللينك — الحاسبة هتشتغل فورًا لأي حد من أي مكان.

> **ملاحظة:** لو الريبو خاص (Private) لازم يكون عندك GitHub Pro علشان GitHub Pages يشتغل، أو خليه Public عادي (البيانات مش حساسة زي كلمة سر، هي أسعار منتجات).

## 2) إعداد صلاحية الأدمن (اسم مستخدم وكلمة مرور + الحفظ المباشر على GitHub)

### تسجيل الدخول باسم المستخدم وكلمة المرور
بيانات الدخول الافتراضية:
- **اسم المستخدم:** `admin`
- **كلمة المرور:** `admin123`

**⚠️ غيّرها فورًا بعد أول رفع** من تبويب لوحة الأدمن → كارت "تغيير بيانات دخول الأدمن" → اكتب اسم مستخدم وكلمة مرور جديدين → اضغط "تحديث بيانات الدخول" → ثم لازم تضغط "حفظ التعديلات على GitHub" (أو تنزيل JSON ورفعه يدويًا) عشان التغيير يتفعّل فعليًا لكل الأجهزة.

**⚠️ تنبيه أمان مهم:** الموقع Static (مفيهوش سيرفر)، فكلمة المرور بيتم التحقق منها في المتصفح فقط، وده معناه:
- كلمة المرور بتتحفظ كـ **hash (SHA-256)** جوه `data.json` مش كنص صريح، لكن لو الريبو **Public** فأي حد يقدر يشوف الـ hash ده (زي ما بيشوف أي بيانات تانية في data.json) ويحاول يكسره بمحاولات كتير (brute force) خصوصًا لو كلمة المرور بسيطة أو قصيرة.
- عشان كده: **استخدم كلمة مرور قوية وطويلة**، أو خلي الريبو **Private** لو حابب حماية إضافية (يتطلب GitHub Pro/Team عشان GitHub Pages يشتغل مع الريبوهات الخاصة، زي ما موضح تحت).
- تسجيل الدخول باسم المستخدم وكلمة المرور ده طبقة حماية إضافية بس *قبل* الوصول للوحة الأدمن؛ الحفظ الفعلي على GitHub لسه بيحتاج GitHub Token زي ما هو موضح تحت (طبقة تانية منفصلة).

### الحفظ الفعلي على GitHub (طبقة تانية بعد الدخول)
بعد ما تدخل باسم المستخدم وكلمة المرور، لسه محتاج **GitHub Token** بصلاحية الكتابة على الريبو عشان تقدر تحفظ التعديلات وتظهر لكل الزوار — التوكن ده هو المفتاح الحقيقي للكتابة على GitHub API، وده منفصل تمامًا عن اسم المستخدم/كلمة المرور. تقدر تلاقي حقول التوكن تحت "إعدادات الاتصال بـ GitHub" في شاشة الدخول (قابلة للطي، تتملى مرة واحدة وتتذكر في المتصفح لو فعّلت "تذكرني").

### إنشاء التوكن (مرة واحدة):
1. روح على GitHub → صورتك الشخصية (أعلى اليمين) → **Settings**
2. من القائمة الجانبية: **Developer settings → Personal access tokens → Fine-grained tokens**
3. **Generate new token**
4. الإعدادات:
   - **Repository access:** Only select repositories → اختار الريبو بتاع الموقع فقط (مش كل الريبوهات بتاعتك)
   - **Permissions → Repository permissions → Contents:** اختار **Read and write**
   - أي صلاحية تانية سيبها زي ما هي (No access)
5. حدد تاريخ انتهاء (Expiration) مناسب — مثلاً 90 يوم أو سنة، وجدد لما ينتهي.
6. **Generate token** وانسخه فورًا (مش هيظهر تاني).

### استخدام التوكن في الموقع:
1. افتح الموقع → تبويب **لوحة الأدمن**
2. دخّل:
   - **Owner:** اسم المستخدم بتاعك على GitHub
   - **Repo:** اسم الريبو
   - **Branch:** غالبًا `main`
   - **Token:** التوكن اللي عملته
3. اضغط **دخول**. لو نجح، هتلاقي كل الجداول قابلة للتعديل.
4. بعد أي تعديل (سعر لوح، موديل إنفرتر، خصم...) اضغط **"حفظ التعديلات على GitHub"** — هيعمل commit تلقائي لملف `data.json` في الريبو، ويظهر التحديث لكل زوار الموقع خلال ثواني.

### ⚠️ تنبيهات أمان مهمة
- **متشاركش التوكن مع حد مش موثوق فيه** — أي حد معاه التوكن يقدر يعدل أسعارك.
- خيار "تذكرني على هذا الجهاز" بيحفظ التوكن في متصفحك (localStorage) — استخدمه بس على جهازك الشخصي، وامسحه لو الجهاز مشترك (زرار "خروج" بيمسحوش تلقائي، امسح بيانات المتصفح لو محتاج).
- التوكن مربوط بريبو واحد بس (Fine-grained) فحتى لو اتسرب، الضرر محصور في الريبو ده.
- لو عايز أكتر من أدمن، كل واحد يعمل توكن خاص بيه بنفس الطريقة.

### بديل بدون توكن (للتجربة السريعة)
زرار **"تعديل بدون حفظ فوري"** بيسمحلك تعدل الأسعار وتشوف النتيجة، وتنزل ملف `data.json` المُعدَّل يدويًا، وترفعه بنفسك على GitHub. مفيد لو مش عايز تعمل توكن دلوقتي.

## 3) هيكل بيانات `data.json` (لو حبيت تعدل يدوي بمحرر نصوص)

أهم الأقسام:
- `panels`: كل لوح شمسي — `brand, power, vimp, voc, iimp, isc, price` (السعر بالجنيه لكل واط)
- `inverter.discounts` + `inverter.models`: خصومات الماركات وموديلات الإنفرتر (`hp, kw, listPrice`)
- `combinerBox.table`: صناديق التجميع حسب عدد المصفوفات
- `reactor.table` / `mcb`: أسعار الريأكتور والقواطع
- `steel`: أسعار الحديد (ثابت / متحرك سيمكو / متحرك الموحدة)
- `cables`, `earthing`, `installation`: كل الثوابت التانية (أسعار المتر، هامش الربح، التركيب، النقل...)

كل الحقول دي متاحة كمان من لوحة الأدمن نفسها بدون الحاجة لتعديل الملف يدويًا.

---

## 4) سجل عروض الأسعار (تسجيل مركزي لأي عميل/مندوب/أدمن)

أي عرض سعر حد يعمله ويطبعه/يحفظه بيتسجل تلقائيًا:
- **محليًا على جهازه فورًا** (احتياطي، شغال دايمًا من غير أي إعداد).
- **مركزيًا** بحيث الأدمن يقدر يشوفه من أي جهاز - وده محتاج إعداد لمرة واحدة بس:

1. افتح `server/quote-logger-worker.js` واتبع خطوات النشر المكتوبة في أول الملف (نشر مجاني على Cloudflare Workers، ٥ دقايق تقريبًا).
2. بعد النشر، هيديك Cloudflare رابط زي: `https://alasl-quote-logger.YOUR-SUBDOMAIN.workers.dev`
3. حط الرابط ده في لوحة الأدمن → كارت "سجل عروض الأسعار" → حقل "رابط سجل عروض الأسعار (Worker URL)" → احفظ التعديلات على GitHub.

من ساعتها، أي عرض (من عميل عادي، مندوب، أو أدمن) هيتسجل في `quotes.json` على الريبو تلقائيًا، وهيظهر لأي أدمن يسجل دخول من أي جهاز، بنفس اسم المستخدم وكلمة المرور بتوع لوحة الأدمن.

**ليه الطريقة دي بالذات؟** التوكن الحقيقي اللي بيقدر يكتب على الريبو بيفضل مخبأ جوه إعدادات الـ Worker على Cloudflare، ومش بيظهر خالص في كود الموقع اللي أي زائر يقدر يشوفه - فمفيش خطر إن حد يستخدمه بشكل غلط.
